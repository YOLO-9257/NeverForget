
# 智能管家 Agent 升级指南 (True Agentic Butler)

## 概述
v1.3.0 带来的最大更新是将“AI 智能管家”从单纯的聊天机器人升级为具备行动能力的 **AI Agent**。
它现在可以理解您的意图，并直接调用系统内部的工具来执行操作，例如查询数据、创建任务和管理配置。

## 核心能力
得益于全新的 **Function Calling (工具调用)** 架构，智能管家现在支持以下原生能力：

### 1. 任务管理
*   **创建提醒**: 您可以直接说 "明天早上8点提醒我开会"，Agent 会自动解析时间并创建任务。
    *   *智能配置复用*: 如果您之前创建过任务，Agent 会自动复用您的推送配置 (AppID/AppSecret)。
*   **查询任务**: "我还有哪些未完成的任务？" 或 "看看我有几个活跃的提醒"。

### 2. 系统洞察
*   **系统报告**: "系统运行状态怎么样？" Agent 会返回当前的任务总数、活跃数以及最近的执行日志。

### 3. 配置管理 (模板支持)
*   **保存配置**: "把这个模板ID保存为'早安日报'"。
*   **列出配置**: "我有哪些保存的模板？"

## 技术实现细节

### 架构变更
我们引入了经典的 Agent Loop (智能体循环) 模式：
1.  **Thinking**: LLM 接收用户消息，决定是否需要使用工具。
2.  **Act**: 如果需要工具，Worker 会拦截请求并在本地数据库执行 (无需回调外部)。
3.  **Observe**: 工具执行结果被送回给 LLM。
4.  **Response**: LLM 根据工具结果生成最终回复。

### 代码结构
*   `src/utils/aiTools.ts`: 定义了所有可用工具 (`query_reminders`, `create_reminder`, `save_config`, etc.)。
*   `src/utils/aiClient.ts`: 升级了 API 客户端，支持 Gemini 和 OpenAI 的 Native Tools 标准。
*   `src/handlers/aiChat.ts`: 实现了多轮对话循环，支持最大 5 次连续工具调用。

## 使用示例

#### 示例 1: 创建简单任务
> **User**: 明天上午10点提醒我给客户发邮件。
> **Agent**: 没问题，我已经创建了提醒“给客户发邮件”，将在明天 10:00 触发。[调用 create_reminder]

#### 示例 2: 复杂查询
> **User**: 系统最近有报错吗？
> **Agent**: (调用 get_system_report) 查看了最近的日志，系统运行正常，没有发现失败记录。最近一次任务在 10 分钟前成功执行。

#### 示例 3: 管理配置
> **User**: 记住我的 WxPush UserID 是 UID_123456，名字叫“主账号”。
> **Agent**: (调用 save_config) 好的，已保存配置“主账号”。

## 注意事项
*   Agent 的能力受限于 LLM 的推理能力。推荐使用 **Gemini 2.0 Flash** 或 **GPT-4o** 以获得最佳体验。
*   由于 Worker 执行时间限制，极为复杂的连续对话可能会遇到超时（虽然我们限制了最大 5 轮）。
